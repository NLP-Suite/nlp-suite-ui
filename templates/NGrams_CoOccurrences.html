In INPUT the script expects a single txt file or a directory containing a set of txt files.
Please, select the directory where the script will save all OUTPUT files of any type (txt, csv, png, html).
MOST GUI SCRIPTS WILL ORGANIZE OUTPUT FILES IN SPECIFIC SUBDIRECTORIES INSIDE THE MAIN OUTPUT DIRECTORY SELECTED HERE (e.g., the SVO GUI will create several subdirectories: GIS, SVO, SVO-filtered, SVO-unfiltered, SVO-lemma, WordNet).
Please, tick the checkbox if you wish to compute N-grams.

N-grams can be computed for character and word values. Use the dropdown menu to select the desired option.

N-grams frequencies are exported by document and by corpus in all case-sensitive variations (e.g., It is necessary, it Is necessary, It is Necessary,...).

In OUTPUT, the script generates a set of csv files each containing word N-grams between 1 and 3.

When N-grams are computed by sentence index, the sentence displayed in output is always the first occurring sentence. Select the N-grams type.
Please, use the dropdown menu to select different options for N-grams. Click the + button to select more options; the Reset button to start fresh.

In OUTPUT, the script generates the following three files:
1. csv file of frequencies of the twenty most frequent words;
2. csv file of the following statistics for each column in the previous csv file and for each document in the corpus: Count, Mean, Mode, Median, Standard deviation, Minimum, Maximum, Skewness, Kurtosis, 25% quantile, 50% quantile; 75% quantile;
3. Excel line chart of the number of sentences and words for each document. Select the number of N-grams to be computed. 1-grams and hapax legomena (once-occurring words) are always computed automatically.       Select the N-grams option; hit + button to add multiple options; Reset to start fresh.
The case-sensitive option will display the sum of all different cases and the individual cases (e.g., It is, it Is, It Is).         
Visualize the options already selected.  
Please, enter the comma-separated list of single words or collocations (multi-word expressions) (i.e., sets of words such as coming out, beautiful sunny day) for which you want to know N-Grams/Co-occurrences statistics (e.g., woman, man, job). Leave blank if you do not want NGrams data. Both NGrams and co-occurrences words can be entered. Enter the integer number of words (do not enter -) preceding the search word to be extracted, for context, together with the search sentences.
The option does not apply to N-grams VIEWER or Co-Occurrences VIEWER. Enter the integer number of words (do not enter +) following the search word to be extracted, for context, together with the search sentences.
The option does not apply to N-grams VIEWER or Co-Occurrences VIEWER.
Please, use the dropdown menu to select various options that can be applied to the VIEWER. Multiple criteria can be seleced by clicking on the + button.

You can make your searches CASE SENSITIVE.

You can NORMALIZE results. Only works for N-Grams. Formula: search word frequency / total number of all words e.g: word "nurse" occurs once in year 1892, and year 1892 has a total of 1000 words. Then the normalized frequency will be 1/1000.

You can SCALE results. Only works for N-Grams. It applies the min-max normalization to frequency of search words. After the min-max normalization is done, each column of data (i.e., each search word) will fall in the same range.

You can LEMMATIZE words for your searches (e.g., be instead of being, is, was). The routine relies on Stanza for lemmatizing words. WHEN LEMMATIZING THE INPUT TEXT, SEARCH WORDS WILL ALSO BE LEMMATIZED.

Finally, you can select to display minimal information or full information.

Please, tick the checkbox to search for selected words in any output csv file produced by the 'Compute N-grams' algorithm.
Upon clicking you will be prompted to select the N-grams csv file to be used.   
Please, tick the Ngram VIEWER checkbox if you wish to run the Ngram Viewer script.

Tick the Co-Occurrence VIEWER checkbox if you wish to run the Co-Occurrence Viewer script.

You can run both Viewers at the same time.

The NGrams part of the algorithm requires date metadata, i.e., a date embedded in the filename (e.g., The New York Time_2-18-1872).

For both viewers, results will be visualized in Excel line plots.

For N-grams the routine will display the FREQUENCY OF NGRAMS (WORDS), NOT the frequency of documents where searched word(s) appear.

For Word Co-Occurrences the routine will display the FREQUENCY OF SENTENCES or DOCUMENTS where searched word(s) appear, depending upon the selected search option (within sentence or within document). The N-grams VIEWER option requires in input file(s) with a date embedded in the filename. Depending upon the selected search options, you can search for co-occurrences within sentences or within documents.
The Co_Occurrences VIEWER option does NOT require in input file(s) with a date embedded in the filename, contrary to the N-grams VIEWER.       Tick the checkbox to search co-occurring words within sentence (default: within document only).
The option applies to Co-Occurrences VIEWER only.
Please, tick the checkbox if the filenames embed a date (e.g., The New York Times_12-19-1899). The DATE OPTIONS are required for N-grams; optional for word co-occurrences.

Please, using the dropdown menu, select the level of temporal aggregation you want to apply to your documents: group of years, year, quarter, month.
