Please, tick the checkbox to run the Stanford CoreNLP coreference resolution annotator using the Neural Network approach. Only pronominal, and not nominal, coreference resolution is implemented for four different types of PRONOUNS: nominative: I, you, he/she, it, we, they; possessive: my, mine, our(s), his/her(s), their, its, yours; objective: me, you, him, her, it, them; reflexive: myself, yourself, himself, herself, oneself, itself, ourselves, yourselves, themselves. Please, BE PATIENT. Depending upon size and number of documents to be coreferenced the algorithm may take a long a time. In INPUT the algorithm expects a single txt file or a directory of txt files. In OUTPUT the algorithm will produce txt-format copies of the same input txt files but co-referenced. Tick the checkbox to run the pronominal coreference resolution and run the SVO extractor on coreferenced files.
Please, tick the checkbox if you wish to resolve manually cases of unresolved or wrongly resolved coreferences. The option is not available when processing a directory of files. You can always use the 'coreference_main' GUI to 1. open a merged coreferenced file; 2. split merged coreferenced files. If manual edit is selected, the script will also display a split-screen file for manual editing. On the left-hand side, pronouns cross-referenced by CoreNLP are tagged in YELLOW; pronouns NOT cross-referenced by CoreNLP are tagged in BLUE. On the right-hand side, pronouns cross-referenced by CoreNLP are tagged in RED, with the pronouns replaced by the referenced nouns.\n\nMANUAL EDITING REQUIRES A LOT OF MEMORY SINCE BOTH ORIGINAL AND CO-REFERENCED FILE ARE BROUGHT IN MEMORY. DEPENDING UPON FILE SIZES, YOU MAY NOT HAVE ENOUGH MEMORY FOR THIS STEP. Tick the checkbox to manually edit a coreferenced file fixing missed (or wrongly) coreferenced pronouns. Manual coreference is available only when coreferencing a single input document.

The three widgets display the currently selected dictionary filter files for Subjects, Verbs, and Objects (Objects share the same file as Subjects and you may wish to change that). The filter file social-actor-list, created via WordNet with the multiple keywords act, group, person and saved in the \'lib/wordLists\' subfolder, will be automatically set as the DEFAULT filter for subjects (Press ESCape to clear selection); the file \'social-action-list.csv\' is similarly created via WordNet using multiple keywords (change, cognition, communication, contact, emotion, motion, social), saved in the \'lib/wordLists\' subfolder, and automatically loaded as the DEFAULT dictionary file for verbs. The widgets are disabled because you are not allowed to tamper with these values. If you wish to change a selected file, please tick the appropriate checkbox in the line above (e.g., Filter Subject) and you will be prompted to select a new file. Upload csv file containing SUBJECT filters:
Please, tick the 'Lemmatize' checkboxes to produce lemmatized subjects. When SVOs are lemmatized, the algorithm will aggregate the Subjects into WordNet top synset categories (e.g., 'run' into 'motion'). Tick the 'Filter' checkboxes to filter all SVO extracted triplets for Subjects via dictionary filter files. When lemmatizing subjects, WordNet will be used to aggregate subjects into top synsets noun categories. Filter subjects list excluding subjects that are not social actors. When S and V and O filters are selected all conditions must be met. The option for filtering subjects via WordNet for social actors is available only for the English language. Words in MWE, except entitymentions, are _ separated.
Upload csv file containing VERB filters:
Please, tick the 'Lemmatize' checkboxes to produce lemmatized verbs. When SVOs are lemmatized, the algorithm will aggregate the Verbs into WordNet top synset categories (e.g., 'run' into 'motion'). Tick the 'Filter' checkboxes to filter all SVO extracted triplets for Verbs via dictionary filter files. When lemmatizing verbs, WordNet will be used to aggregate subjects into top synsets verb categories. Filter verbs list excluding verbs that are not social actions. When S and V and O filters are selected all conditions must be met. The option for filtering verbs for social actions via WordNet for social actors is available only for the English language. Words in MWE are _ separated.
Upload csv file containing OBJECT filters:
Please, tick the 'Lemmatize' checkboxes to produce lemmatized objects. When SVOs are lemmatized, the algorithm will aggregate the Objects (nouns) into WordNet top synset categories (e.g., 'run' into 'motion'). Tick the 'Filter' checkboxes to filter all SVO extracted triplets for Objects via dictionary filter files. When lemmatizing objects, WordNet will be used to aggregate objects into top synsets object categories. Filter objects list excluding objects that are not social actors. When S and V and O filters are selected all conditions must be met. The option for filtering objects for social actors via WordNet for social actors is available only for the English language. Words in MWE, except entitymentions, are _ separated.
Please, tick the S & O gender checkbox if you wish to run Stanford CoreNLP neural network gender annotator to extract the gender (female, male) for every Subject and Object extracted by the SVO script. Tick the S & O quote/speaker checkbox if you wish to run Stanford CoreNLP neural network quote annotator to extract the speaker involved in direct discourse for every Subject and Object extracted by the SVO script. THE GENDER AND QUOTE/SPEAKER ANNOTATORS ARE AVAILABLE FOR STANFORD CORENLP AND ENGLISH LANGUAGE ONLY. The neural network gender annotator is available only via Stanford CoreNLP and for the English language only. The neural network quote annotator is available only via Stanford CoreNLP.
